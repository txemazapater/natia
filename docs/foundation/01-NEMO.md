# NEMO

> NEMO no es un proyecto. Es una propiedad.

Este documento fundacional define NEMO como la capacidad de un Workspace o proyecto para preservar, explicar y reutilizar la evolución de su conocimiento.

## Principios

- NEMO no es una aplicación.
- NEMO no es un servicio.
- NEMO no es una base de datos.
- NEMO es una propiedad de un Workspace.

## La pregunta fundamental

El objetivo de NEMO no es responder «¿qué sabemos?», sino «¿cómo hemos llegado hasta aquí?».

Las conversaciones dejan de ser un historial para convertirse en evidencia del razonamiento.
